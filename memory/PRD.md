## Overview
Standalone **native Android (Kotlin)** application whose only purpose is to push
GPS coordinates to a configurable HTTP endpoint at a chosen interval. Designed
to be installed on a child or senior's device by a parent/guardian. Completely
independent from the *My Family My Life* web project.

> Source path: `/app/family-location-sender/` (NOT the Expo template at `/app/frontend`).

## Stack
- **Language:** Kotlin 1.9.24
- **Min SDK:** 24 (Android 7.0)  /  **Target SDK:** 34 (Android 14)
- **Gradle:** 8.7 (wrapper auto-generated by CI)
- **UI:** Material 3 + ViewBinding (XML layouts)
- **Background:** Foreground Service + FusedLocationProvider
- **Networking:** OkHttp 4.12 (plain HTTP/HTTPS, JSON via `org.json`)
- **Storage:** AndroidX Security `EncryptedSharedPreferences` (AES-256)

## Features delivered
1. **Password gate** with default `1001`, configurable; SHA-256 hashed in
   encrypted prefs. `FLAG_SECURE` on lock screen.
2. **Auto-lock** on background, screen off, app switch, or 1 min idle.
3. **First-run Setup screen** collects name, family code, API endpoint,
   profile photo (camera/gallery), interval, optional new password.
4. **Main Dashboard** shows live GPS / network / connection type / battery /
   last send / success counter / failure counter / service status / current
   API. Buttons: Start, Stop (password), Settings (password), Test, Refresh,
   Lock Now.
5. **Settings screen** (password-protected) â€” change every setting + password
   + enable Device Admin + switch language.
6. **Foreground Service** with persistent notification, `foregroundServiceType="location"`.
7. **Smart Mode**: 1 min when moving, 15 min when stationary, immediate send on
   >100 m displacement. Also fixed 1/3/5/15 min options.
8. **Boot Receiver** restarts tracking on reboot / app upgrade.
9. **Device Admin** receiver (`force-lock`, `watch-login`, `reset-password`)
   for soft uninstall protection. Warning card on dashboard if not enabled.
10. **Bilingual** â€” English (default) + Arabic (RTL); switchable at runtime.
11. **JSON payload** matches the spec from the brief.
12. **GitHub Actions workflow** that builds `app-debug.apk` (and unsigned
    release APK) on every push.
13. **Offline queue** (encrypted, FIFO, capped at 500 entries) â€” failed/offline
    payloads are persisted and auto-replayed when connectivity returns
    (via `ConnectivityManager.NetworkCallback`). Drained items are deleted
    immediately â€” **no on-device history**.
14. **Battery-optimization warning card** on the dashboard with one-tap exempt
    button, shown only when the app is still subject to optimization.

## What is **not** in the app (by design)
- No SOS / panic feature
- No medical / health features
- No in-app map
- No on-device location history (only counters + last-send timestamp)
